# Daily Digest

Утренний дайджест "кому писать сегодня".

## Команды

```
/daily-digest                          # просто показать
/daily-digest --post-to-github 6       # запостить в issue #6
/daily-digest --post-to-telegram       # в личку через Telethon
/daily-digest --dry-run                # без записи
```

## Что делает

### 1. Чтение CRM
Тянет все активные листы (Лиды, Сделки, Клиенты — не Беклог). Игнорирует помеченные ⚫ закрытыми.

### 2. Анализ каждой строки
Параллельно по 20-30 строк через subagent (Haiku — быстрый и дешёвый).

Для каждой:
- Извлекает последнюю дату из "Истории взаимодействия"
- Считает дни до сегодня
- Определяет срочность:

| Срочность | Условия |
|---|---|
| 🔴 Высокая | Горячий лид + >14 дней молчания / КП отправлено >5 дней без реакции / "ждём ответа" >7 дней |
| 🟡 Средняя | Пауза 14-30 дней / нужно проактивно вернуться |
| 🟢 Низкая | Недавно общались / "не заинтересован" / пауза по плану |
| ⚫ Закрыто | Отказ / выиграно / не работаем |

### 3. Черновики для красной зоны
Для каждого 🔴 — короткое сообщение в стиле из `.env`:
- Прямой: "Привет, X. По нашему КП — есть конкретные вопросы?"
- Тёплый: "Привет, X! Как дела? Думал про тебя — давно не общались..."
- Формальный: "Здравствуйте, X. Уточните, пожалуйста, статус по нашему предложению..."

Контекст для черновика берётся из "Истории взаимодействия" — что было последним.

### 4. Дайджест

Markdown:

```markdown
## ☀️ CRM-дайджест на 2026-MM-DD

### 🔴 Красная зона (X сделок)
1. **Компания A** — N дней молчания
   _Контекст_: что было последним
   _Action_: что предлагаю сделать
   ```
   <готовый черновик сообщения>
   ```

### 🟡 На неделе (X сделок)
- Компания B — N дней
- ...

### 🟢 Поддерживать (X сделок)
- ...

### 📊 Метрики
- Всего активных: ...
- Новых за неделю: ...
- Закрыто за неделю: ...
```

### 5. Публикация (опционально)
- В GitHub issue (если `--post-to-github N`)
- В Telegram личку (если `--post-to-telegram` и есть TG_SESSION)
- В Google Doc (если `--save-to-drive`)

## Конфиг

`.env`:
```env
DIGEST_TIMEZONE="Europe/Moscow"
DIGEST_RED_ZONE_DAYS=14         # порог дней для красной зоны
DIGEST_KP_GRACE_DAYS=5          # сколько ждать после КП
DIGEST_GENERATE_DRAFTS=true     # генерить ли черновики (стоит токенов)
DIGEST_POST_DEFAULT="github"    # github|telegram|none
```

## Когда автоматизировать

Через remote agent (`/schedule`):

```
cron: "0 6 * * 1-5"   # пн-пт 09:00 МСК = 06:00 UTC
command: /daily-digest --post-to-github <issue-number>
```

## Принципы

1. **Не доверять только тексту в CRM** — делать пометку "проверь актуальный статус"
2. **Не повторять черновики** — если за прошлый раз сгенерили похожий, диверсифицировать
3. **Учитывать выходные** — не считать дни 1 в 1, для пятницы напоминание про метрики недели
4. **Учитывать праздники** (если задано) — не пинговать в нерабочие дни

## Связанные скиллы

- `/sync-messenger --all` — запускать ДО `/daily-digest` (свежие переписки попадут в анализ)
- `/meeting-notes` — для дополнения "Истории" свежими расшифровками
- `/add-to-crm` — для приоритизации новых лидов в дайджесте
