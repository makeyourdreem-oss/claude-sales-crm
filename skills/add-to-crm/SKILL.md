# Add to CRM

Универсальная команда: что-то узнал про компанию — кидаешь в Claude → попадает в CRM.

## Использование

```
/add-to-crm <URL>
/add-to-crm <путь/к/файлу>
/add-to-crm "вставленный текст"
/add-to-crm @<контакт-в-чате>     # парсит контекст разговора
```

## Что делает (pipeline)

### 1. Парсинг входа
- URL → fetch контента (WebFetch → fallback Jina Reader)
- .docx / .pdf → Read tool / python-docx / pdfminer
- Просто текст → принять как есть
- `@контакт` или имя → собрать всё что упомянуто в текущем чате

### 2. Извлечение сущностей
Из контента вытащить:
- Названия компаний
- ФИО людей + должности
- Контакты (email, телефон, Telegram)
- Описание продукта/деятельности
- Связи между сущностями ("X работает в Y")

### 3. Классификация
Для каждой компании определить тип:
- **Лид** — потенциальный клиент по ICP
- **Конкурент** — продают то же или похожее
- **Партнёр** — может рекомендовать / интегрироваться
- **Поставщик** — мы можем у них купить
- **Прочее** — не классифицировано

Для людей определить: `decision_maker | influencer | gatekeeper | champion | партнёр-связной`.

### 4. Авто-обогащение (если данных мало)
Если про компанию известно только название:
- Каскад поиска: первый рабочий из `WEB_SEARCH_CASCADE`
- Извлечь: отрасль, чем занимаются, регион, сайт, размер
- Если не нашёл — пометить `[нужно проверить]`, не выдумывать

### 5. Дедуп
Перед записью — поиск по названию компании во всех CRM-листах:
- Если найдено — обновить существующую строку (append в "Историю взаимодействия", не перезаписывая)
- Если нет — создать новую строку в подходящем листе

### 6. Запись
- Сразу без подтверждения (если иначе не настроено в `.env`: `CONFIRM_BEFORE_WRITE=true`)
- В CRM лог: `[YYYY-MM-DD]: добавлено через /add-to-crm из <источник>`

## Маппинг тип → лист

| Тип | Лист |
|---|---|
| Лид (холодный) | Лиды |
| Лид (после первого касания) | Сделки |
| Клиент (закрытый) | Клиенты |
| Партнёр | Партнёры |
| Конкурент | Конкуренты |
| Контакт (человек) | Контакты |
| Прочее | Беклог |

## Важные правила

1. **Не перезаписывать** существующие данные. Только append в "Историю взаимодействия".
2. **Не удалять** строки. Никогда. Service account имеет полные права — это опасно.
3. **При неоднозначности** — спросить пользователя (например, две компании с похожим названием).
4. **Не выдумывать** контактные данные. Если email не найден — оставить пустым с пометкой.

## Примеры

### Кейс 1: добавить компанию по URL
```
/add-to-crm https://example.com
```
→ парсит сайт → определяет: "Лид, software, Москва" → добавляет в **Лиды**

### Кейс 2: расшифровка встречи
```
/add-to-crm путь/к/расшифровке-встречи.docx
```
→ извлекает упомянутые компании и людей → создаёт/обновляет несколько строк → ссылается на расшифровку в "История"

### Кейс 3: ссылка на конкурента
```
/add-to-crm https://srizfly.com — конкурент
```
→ парсит → классифицирует как конкурент (явно указано) → добавляет в **Конкуренты** с инфой о ценах/продукте
