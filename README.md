# Claude Sales CRM Toolkit

> Личный AI-ассистент для B2B-продаж: ведёт CRM в Google Sheets, синхронизирует переписки из мессенджеров, превращает расшифровки звонков в саммари, готовит ежедневные дайджесты "кому писать сегодня".

Набор скиллов для **Claude Code**, который делает за тебя рутину B2B-продавца:

- 📊 Ведёт CRM в Google Sheets (без боли с протухающими OAuth-токенами — на service account)
- 💬 Тянет переписки из Telegram → саммари → апдейты в CRM
- 🎙️ Обрабатывает расшифровки встреч (Яндекс Конспект, Whisper, любой текст) → структурирует в DOCX (транскрипция / саммари / выводы)
- ☀️ Утренний дайджест: красная зона, кому пинговать, шаблоны сообщений
- ➕ Команда `/add-to-crm` — кидаешь URL/файл/текст → парсит, обогащает через web search, добавляет в нужный лист

Работает с **любым продуктом и любым ICP** — при первом запуске мастер задаст вопросы и подстроит под тебя.

---

## Кому подойдёт

- B2B-продавцы, ведущие сделки руками
- SDR/BDR, у которых много холодного аутрича
- Фаундеры, продающие сами
- Консультанты с долгим циклом сделки
- Все, кому надоело копировать заметки между ТГ, Notion и Google Sheets

## Что внутри

| Скилл | Назначение |
|---|---|
| `/crm-init` | Интерактивный онбординг — создаёт CRM-таблицу под тебя, настраивает доступы |
| `/add-to-crm` | Универсальный inbox: текст / URL / .docx / .pdf / расшифровка → строка в CRM |
| `/meeting-notes` | Расшифровка → структурированный DOCX (транскрибация / саммари / выводы) |
| `/sync-messenger` | Telegram (другие мессенджеры — следующая итерация) → CRM-обновления |
| `/daily-digest` | Утренний дайджест с красной зоной и черновиками сообщений |

Плюс отдельный сервис:

| Сервис | Назначение |
|---|---|
| [`tg-bot/`](tg-bot/) | Личный Telegram-бот: шлёшь голосовое/текст/файл → Gemini парсит → превью с кнопкой подтверждения → запись в CRM. Деплой на VPS через Docker |

## Quick Start

### 1. Установка

```bash
git clone https://github.com/makeyourdreem-oss/claude-sales-crm.git
cd claude-sales-crm
./install.sh
```

`install.sh` копирует скиллы в `~/.claude/skills/` и ставит зависимости.

### 2. Первый запуск

В Claude Code набери:

```
/crm-init
```

Мастер задаст вопросы:
- Что продаёшь? Кому?
- Где хранить CRM? (Google Sheets / Notion / локальный CSV)
- Какие мессенджеры использовать?
- Стиль сообщений? (прямой / тёплый / формальный)

И создаст:
- Google Sheets с готовыми вкладками (Клиенты / Лиды / Партнёры / Конкуренты / Беклог)
- `.env` файл с твоими настройками
- `contacts.json` с шаблоном приоритетных контактов

### 3. Использование

```
/add-to-crm <ссылка-на-сайт-компании>
/add-to-crm путь/к/документу.docx
/add-to-crm "вставленный текст письма"

/meeting-notes путь/к/расшифровке.docx
/meeting-notes "сырой текст расшифровки"

/sync-messenger Илья       # синк одного контакта
/sync-messenger --all      # синк всех приоритетных

/daily-digest              # дайджест на сегодня
```

- [Claude Code](https://claude.com/claude-code) (актуальная версия)

## Архитектура (зачем нужен service account)

OAuth-токены протухают. Это раздражает. Service account — JSON-ключ, который **не протухает**, выдаётся Google проекту, и его можно ограничить только нужной таблицей.

```
┌─────────────┐   service account JSON   ┌──────────────────┐
│ Claude Code │ ───────────────────────▶ │  Google Sheets   │
└─────────────┘                          └──────────────────┘
       │                                          ▲
       │                                          │
       ▼                                          │
┌─────────────┐    ┌──────────────┐    ┌──────────┴───────┐
│  Telegram   │ ─▶ │ Telethon     │ ─▶ │ Скилл-orchestrator│
│  (Telethon) │    │ (api_id+hash)│    │                   │
└─────────────┘    └──────────────┘    └───────────────────┘
       ▲                                          ▲
       │                                          │
       └──────────────────────────────────────────┘
                    переписки + расшифровки
```
